1. Can a Single Model Master Both Multi-turn Conversations and Tool Use? CALM: A Unified Conversational Agentic Language Model

  • 作者: Emre Can Acikgoz et al.
  • 机构: University of Illinois Urbana-Champaign, Oumi
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 5
  • 核心方法: CoALM 把 Task-Oriented Dialogue 和 Language Agent 的 function calling 能力统一到一个模型里。构造了 CoALM-IT 多任务数据集，交织多轮 ReAct 推理和复杂 API 调用。训练了 8B/70B/405B 三个尺度，在 MultiWOZ 2.4、BFCL V3、API-Bank 上全面超越 GPT-4o。
  • 对比方法/Baselines: GPT-4o, specialized TOD systems, ToolAce
  • 借鉴意义: 对话式 agent 的核心痛点——多轮对话管理和工具调用不能兼得——这篇给了一个干净的统一方案。对做对话推荐系统的同学直接有用。
  • 锐评: 思路清晰，实验扎实，三个 benchmark 全打是有说服力的。但 405B 的模型谁用得起？8B 版本的性能衰减多少才是关键。另外 CoALM-IT 数据集用 GPT-4o 生成，那你超越 GPT-4o 到底是模型强还是数据蒸馏的功劳？🔥
  • 💡 想精读？ 运行：读一下 CALM: A Unified Conversational Agentic Language Model

    • 2. AgentScope 1.0: A Developer-Centric Framework for Building Agentic Applications

  • 作者: Dawei Gao et al.
  • 机构: Alibaba Group
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 61
  • 核心方法: AgentScope 是一个面向开发者的 multi-agent 框架，支持 MCP 协议、ReAct 模式、分布式部署。核心抽象包括 MsgHub（消息总线）、StateModule（状态管理）、Non-Invasive Monitoring（基于 OpenTelemetry）。
  • 对比方法/Baselines: LangChain, AutoGen, CAMEL
  • 借鉴意义: 如果你要部署大规模 multi-agent 系统，这个框架的工程设计值得参考——特别是分布式和监控部分。
  • 锐评: 阿里出品，工程质量有保障，但框架类论文的通病是——demo 很美，实际用起来全是坑。MCP 支持是加分项。和 LangChain 的对比不够 fair，只挑了对方弱的场景。👀
  • 💡 想精读？ 运行：读一下 AgentScope 1.0

    • 3. Agent Lightning: Train ANY AI Agents with Reinforcement Learning

  • 作者: Yixiu Liu et al.
  • 机构: Unknown
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 139
  • 核心方法: LightningRL 提供一个通用的 RL 训练框架，可以对接任意 agent 框架（LangChain、AutoGen 等）。核心思路是把 agent 的 trajectory 建模为 MDP，用 GRPO 做策略优化。支持 OpenTelemetry 做训练监控。
  • 对比方法/Baselines: AgentOps, AIR
  • 借鉴意义: RL for agent 的基础设施终于有人认真做了。如果你在做 agent 的训练而不是纯 prompting，这个框架值得关注。
  • 锐评: 139 个 upvote 说明社区饥渴程度。"Train ANY AI Agents" 这标题有点大——实际上对 tool use 的 reward 设计还是得你自己搞。但作为 infra 确实填了一个空白。GRPO 用在 agent trajectory 上是否真的比 PPO 好？论文回避了这个问题。🔥

  • 作者: Unknown
  • 机构: Unknown

    5. SKILL0: In-Context Agentic Reinforcement Learning for Skill Internalization

  • 作者: Unknown
  • 机构: Unknown
  • 链接: arXiv | PDF
  • 来源: 📰 HF Daily，⬆️ 32
  • 核心方法: SKILL0 提出 ICRL（In-Context Reinforcement Learning），让 agent 在上下文中通过 RL 学习技能并内化。使用 SkillBank 存储可复用技能，GRPO 优化。与 ExpeL、MemRL 等 memory-augmented RL 方法对比。
  • 对比方法/Baselines: ExpeL, MemRL, ZeroSearch, SimpleMem
  • 借鉴意义: 把 RL 和 in-context learning 结合的思路对 agent 长期学习有启发。SkillBank 的设计可以借鉴到推荐系统的 agent 策略复用。
  • 锐评: 概念很性感——agent 自己学技能然后记住。但 "skill internalization" 说白了就是把 few-shot examples 用 RL 选出来。在 HotpotQA 上的提升是否能迁移到更复杂的 agent 场景？存疑。👀
  • 💡 想精读？ 运行：读一下 SKILL0

    • 6. Batched Contextual Reinforcement: A Task-Scaling Law for Efficient Reasoning

  • 作者: Unknown
  • 机构: Unknown
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: BCR（Batched Contextual Reinforcement）提出了一个 task-scaling law：通过批量上下文任务训练，可以用更少的计算量达到同等推理能力。基于 GRPO，在 MATH-500、AIME 等数学推理 benchmark 上验证。
  • 对比方法/Baselines: JustRL, BRUMO, DeepSeek-R1
  • 借鉴意义: Scaling law 的研究对理解 RL 训练效率有基础价值。如果你在训练 reasoning model，这个 task-scaling 的视角可以帮你省算力。
  • 锐评: 又一篇 scaling law 论文，但这次角度确实新——不是模型 scale，而是 task scale。实验在数学推理上做的，能否推广到 agent 任务？论文标题说 "efficient reasoning" 但本质上是在说 "怎么更便宜地做 RL"。诚实点没坏处。👀

    7. Optimizing RAG Rerankers with LLM Feedback via Reinforcement Learning

  • 作者: Unknown
  • 机构: Unknown
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: RRPO（Reinforced Reranker Policy Optimization）用 LLM 的反馈信号通过 RL 优化 RAG 的 reranker。在 HotpotQA、AmbigNQ 上验证，用 GTE 作为基础检索器。
  • 对比方法/Baselines: RankZephyr, DRAGIN, SFT baseline
  • 借鉴意义: reranker 是推荐系统和 RAG 共享的核心组件。用 RL + LLM feedback 优化 reranker 的思路可以直接迁移到推荐场景的精排。
  • 锐评: 思路不复杂但实用：与其 SFT 训练 reranker，不如用 RL 让下游 reader 的效果做 reward。问题是 LLM feedback 本身的成本——你用一个大模型的输出训另一个小模型，成本核算过吗？👀
  • 💡 想精读？ 运行：读一下 Optimizing RAG Rerankers with LLM Feedback

    8. MSA: Memory Sparse Attention for Efficient End-to-End Memory Model Scaling to 10M+

  • 作者: Unknown
  • 机构: Unknown
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 44
  • 核心方法: MSA（Memory Sparse Attention）通过稀疏注意力机制将端到端 memory model 扩展到 10M+ token。结合 KV cache 优化和 RoPE 位置编码。在 NIAH、HotpotQA、NarrativeQA 等长文本任务上评测。
  • 对比方法/Baselines: KaLMv2, MemAgent, RWKV
  • 借鉴意义: 长上下文 memory 是 agent 系统的基础能力。如果你在做需要超长历史的推荐 agent，这个稀疏注意力方案值得了解。
  • 锐评: 10M token 的 memory 听起来很唬人，但实际 benchmark 上比 KaLMv2 提升有限。"Sparse" attention 在这个领域已经被做烂了，novelty 在哪？工程味太重，学术贡献偏弱。💤

    • 9. Mem0: Building Production-Ready AI Agents with Scalable Long-Term Memory

  • 作者: Unknown
  • 机构: Unknown
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 50
  • 核心方法: Mem0 提供一个可扩展的 agent 长期记忆系统，支持 ADD/UPDATE/DELETE/CORRECT 操作。与 RAG 对比，强调结构化记忆管理。在 LOCOMO benchmark 上评测。
  • 对比方法/Baselines: MemGPT, LangMem, ReadAgent, Full-Context
  • 借鉴意义: Agent memory 管理是个实际问题，Mem0 的 CRUD 抽象比较工程友好。
  • 锐评: 50 个 upvote 主要因为开源项目本身的热度。论文本身就是把已有产品写成 paper 的操作——技术深度有限，"production-ready" 这词用得太轻巧。和 MemGPT 的对比只在一个 benchmark 上做，不够 convincing。💤

    • 10. MemOS: A Memory OS for AI System

  • 作者: Unknown
  • 机构: Unknown
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 166
  • 核心方法: MemOS 把 agent memory 抽象为操作系统层级的概念，包括 MemScheduler、MemCube、MemStore 等模块。支持 KV cache 管理和 memory lifecycle。
  • 对比方法/Baselines: RAG, MemGPT, traditional KV cache
  • 借鉴意义: 概念框架有参考价值，但落地需要验证。
  • 锐评: 166 个 upvote，概念很宏大——"Memory Operating System"。但论文读完你会发现，这就是一个 memory management 的 wrapper，离真正的 "OS" 差了十个抽象层。MemScheduler 说白了就是个优先级队列。概念大于实质的典型。💤
