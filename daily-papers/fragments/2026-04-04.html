1. PolySwarm: A Multi-Agent Large Language Model Framework for Prediction Market Trading and Latency Arbitrage

  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
📒 精读笔记: PolySwarm

PolySwarm: A Multi-Agent LLM Framework for Prediction Market Trading

一句话总结：用 50 个多样化 LLM persona 并发评估二元预测市场，通过 confidence-weighted Bayesian 聚合与市场隐含概率结合，加上 quarter-Kelly 仓位管理进行交易。

1. 研究动机与问题定义

核心问题

预测市场（如 Polymarket）中的价格是否真的反映了事件的真实概率？单个 LLM 存在幻觉、过度自信和提示敏感性三大问题，难以可靠预测。

理论基础

  • 有效市场假说：市场价格反映所有可用信息
  • Grossman-Stiglitz 悖论：完全有效的市场不会有人愿意付成本去获取信息
  • 群体智慧：多样化个体的独立判断聚合优于单一专家

2. 方法详解

2.1 多样化 Persona 设计

50 个 LLM agent，每个有不同的分析视角和专业背景： - 政治分析师、经济学家、数据科学家、心理学家等 - 不同风险偏好：保守、中性、激进 - 不同信息侧重：基本面、技术面、情绪面

2.2 群体共识聚合

每个 agent $i$ 输出概率估计 $p_i$ 和置信度 $c_i$。加权聚合：

$$p_{\text{swarm}} = \frac{\sum_{i=1}^{N} c_i \cdot p_i}{\sum_{i=1}^{N} c_i}$$

2.3 Bayesian 混合

将群体共识与市场隐含概率 $p_{\text{market}}$ 结合：

$$p_{\text{final}} = \alpha \cdot p_{\text{swarm}} + (1 - \alpha) \cdot p_{\text{market}}$$

其中 $\alpha$ 由群体共识的一致性程度决定。

2.4 期望价值计算

$$\text{EV} = p_{\text{final}} \cdot (\text{payout} - \text{cost}) - (1 - p_{\text{final}}) \cdot \text{cost}$$

只在 $\text{EV} > \text{threshold}$ 时进行交易。

2.5 跨市场套利检测

使用 KL 散度 和 JS 散度检测相关市场间的定价不一致：

$$D_{\text{KL}}(P \| Q) = \sum_x P(x) \log \frac{P(x)}{Q(x)}$$
$$D_{\text{JS}}(P \| Q) = \frac{1}{2} D_{\text{KL}}(P \| M) + \frac{1}{2} D_{\text{KL}}(Q \| M), \quad M = \frac{P + Q}{2}$$

当 $D_{\text{JS}} > \delta$ 时触发套利信号。

2.6 仓位管理：Quarter-Kelly

Kelly 准则 的保守变体：

$$f^* = \frac{p \cdot b - q}{b}, \quad f_{\text{trade}} = \frac{f^*}{4}$$

其中 $p$ 为估计获胜概率，$q = 1 - p$，$b$ 为赔率。使用 1/4 Kelly 以降低波动率。

2.7 评估指标

  • Brier Score：$\text{BS} = \frac{1}{N} \sum_{i=1}^{N} (p_i - o_i)^2$
  • Log-loss：$\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} [o_i \log p_i + (1-o_i) \log(1-p_i)]$

3.2 关键发现

  • 50 个 persona 的聚合显著优于单模型预测
  • Bayesian 混合优于简单平均
  • Quarter-Kelly 在回撤控制上优于 full Kelly
  • 套利检测模块在 ~15% 的市场对中发现了定价不一致

5. 核心概念

  • 预测市场：基于事件结果进行交易的市场
  • Kelly 准则：最优仓位管理公式
  • KL 散度：衡量概率分布差异
  • Bayesian Inference：贝叶斯推断框架
  • Ensemble Methods：集成学习方法
  • Multi-Agent Systems：多智能体系统

6. 优缺点分析

优点

  1. 思路新颖：50 个 LLM persona 的群体智慧方法很 bold
  2. 理论基础：有效市场假说 + Grossman-Stiglitz 悖论提供了合理动机
  3. 风险管理：Quarter-Kelly + 套利检测是实用的工程设计
  4. 多模型对比：测试了主流 LLM 的预测能力

缺点

  1. 无实盘验证：只有方法论评估，没有 Polymarket 实盘结果
  2. 计算成本：50 个 LLM 调用的成本是否值得这个提升？
  3. Persona 设计：50 个 persona 的多样性如何保证？LLM 的 "角色扮演" 是否真正带来独立视角？
  4. 时效性：预测市场依赖实时信息，LLM 的知识截止问题如何处理？
  5. 样本量：评估的市场数量有限，统计显著性存疑

2. SODA: Semi On-Policy Black-Box Distillation for Large Language Models

📒 精读笔记: SODA

SODA: Semi On-Policy Black-Box Distillation for LLMs

一句话总结：在黑盒 LLM 蒸馏中提出 "Semi On-Policy" 折中——混合使用 on-policy 和 off-policy 数据，平衡训练稳定性和学生模型纠错能力。

1. 研究动机

黑盒蒸馏的两难： - Off-policy（如 SeqKD）：从教师采样，简单稳定，但无法纠正学生固有错误 - On-policy（如 GAD）：从学生采样+对抗训练，能纠错但不稳定且计算量大

2. 方法详解

Semi On-Policy 策略

混合采样：以概率 $\alpha$ 从教师采样，概率 $1-\alpha$ 从学生采样：

$$x_t \sim \begin{cases} p_{\text{teacher}}(\cdot | x_{<t}) & \text{w.p. } \alpha \\ p_{\text{student}}(\cdot | x_{<t}) & \text{w.p. } 1-\alpha \end{cases}$$

蒸馏损失：

$$\mathcal{L}_{\text{SODA}} = -\mathbb{E}_{x \sim p_{\text{mix}}} \left[ \sum_t \log p_{\text{student}}(x_t | x_{<t}) \right]$$

$\alpha$ 可以是固定值或按训练进度退火（先 off-policy 后逐渐 on-policy）。

与 DPO 的区别

SODA 是生成式蒸馏（学生生成 → 教师评分），DPO 是偏好对齐（正负样本对比）。两者互补。

3. 核心概念

  • Knowledge Distillation、SeqKD、GAD、DPO、On-Policy Learning

4. 优缺点

优点: 思路自然，工程实用，计算成本可控 缺点: 创新性有限（mixed policy 不算新），需看实验细节判断贡献大小

3. Provable Multi-Task Reinforcement Learning: A Representation Learning Framework with Low Rank Rewards

  • 作者: Yaoze Guo, Shana Moothedath
  • 机构: Iowa State University
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: MTRL（Multi-Task Representation Learning）用低秩 reward 分解做多任务 RL。假设多个 MDP 共享状态-动作空间和转移概率，只有 reward 不同且具有低秩结构。利用这个结构学习共享表示，理论证明了样本复杂度的改进。
  • 对比方法/Baselines: 单任务 RL, MoM (Moment of Method)
  • 借鉴意义: 多任务 RL 的理论分析对理解 representation learning 的本质有帮助，但实际应用中 "共享转移概率" 的假设太强。
  • 锐评: 纯理论工作，Iowa State 的两人团队。低秩 reward 假设倒是合理，但 "相同转移概率" 在实际多任务场景中几乎不可能成立。缺乏任何实验验证——连 toy problem 都没有——让人怀疑理论 bound 的 tightness。典型的 "证明了一个没人会用的 bound" 的理论文。 🫠

4. Enhancing behavioral nudges with large language model-based iterative personalization

  • 作者: Zonghan Li 等
  • 机构: Tsinghua University, University of Toronto, Beijing Normal University
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: 用 LLM 生成个性化节能建议（behavioral nudge），通过微信迭代推送，做了 RCT（随机对照实验）验证效果。传统 nudge 一刀切，LLM 版本根据用户反馈迭代更新建议。
  • 锐评: 是真的做了 field experiment 的论文，这点值得肯定。但核心贡献在于 "LLM 生成个性化文本" 这件早就不新鲜的事情，方法论创新有限。节电效果的 ITE 估计方法也比较标准。清华环境学院出品，更偏社科而非 AI。 💤
