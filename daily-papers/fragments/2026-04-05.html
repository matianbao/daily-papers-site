1. A Logical-Rule Autoencoder for Interpretable Recommendations

  • 作者: Jinhao Pan, Bowen Wei, Ziwei Zhu
  • 机构: George Mason University
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: 提出 LIA（Logical-rule Interpretable Autoencoder），用逻辑规则替代黑盒隐向量做协同过滤。每个用户的推荐结果可以用 AND/OR 逻辑规则解释（"因为你看了 A 和 B，所以推荐 C"）。基于 MultVAE 架构改造，latent space 被约束为离散逻辑操作。
  • 对比方法/Baselines: MultVAE, MF, 传统 协同过滤
  • 借鉴意义: 可解释推荐一直是工业界的痛点——模型说不清为什么推荐。逻辑规则方法提供了 intrinsic interpretability，比 post-hoc 解释更可信。
  • 锐评: 想法不错，但 George Mason 小组的论文通常实验规模偏小。逻辑规则能不能在大规模稀疏数据上保持表达能力存疑——协同过滤的精髓就是隐式模式发现，强行离散化可能丢信息。另外 NDCG 指标对比 MultVAE 能赢多少？摘要没提具体数字，有点心虚。 👀

2. APPA: Adaptive Preference Pluralistic Alignment for Fair Federated RLHF of LLMs

  • 作者: Mahmoud Srewa, Tianyu Zhao, Salma Elmalaki
  • 机构: UC Irvine
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: APPA 解决联邦 RLHF 中多群体偏好冲突问题。核心是 adaptive preference aggregation——不是简单平均各方 reward，而是动态调整不同群体的权重，确保少数派偏好不被淹没。用 PPO 做 policy optimization，SFT 做初始化。
  • 对比方法/Baselines: PluralLLM, DPA, OQA, GLOBALQA
  • 借鉴意义: 联邦场景下的 RLHF 对齐是实际部署中越来越重要的问题——不同地区/文化的用户对"有用"和"安全"的定义天差地别。
  • 锐评: 方向没问题，但 "pluralistic alignment" 本身就是个模糊概念——如何量化"公平"？不同群体的 reward 加权怎么选？如果某个群体的偏好本身就有问题（比如要求模型输出有害内容），"公平"的定义就需要重新审视。理论框架不错，但实际效果取决于群体划分的粒度。 👀

3. Combee: Scaling Prompt Learning for Self-Improving Language Model Agents

  • 作者: Hanchen Li 等
  • 机构: UC Berkeley, Stanford, Tensormesh, Gradient Network
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: Combee 把 prompt learning 从单 agent 扩展到大规模多 agent 场景。之前的方法如 ACE 和 GEPA 只在单 agent 或低并行度下有效，Combee 通过分层 prompt 结构和共享 prompt bank 实现 scaling——多个 agent 共享学习到的 prompt 知识。
  • 对比方法/Baselines: ACE, GEPA
  • 借鉴意义: Inference-time learning 不改参数只改 prompt 的思路越来越成熟，scaling 到多 agent 是自然的下一步。
  • 锐评: Berkeley + Stanford 的阵容看着强，但 prompt learning 这个方向本身的天花板在哪里？不改参数的学习终究受限于 prompt 的表达能力。另外 "self-improving" 这个词用得太大了——到底能自我改进到什么程度？如果只是在固定 benchmark 上 prompt search，那跟 AutoML 换皮差不多。 👀

4. TimeSeek: Temporal Reliability of Agentic Forecasters

  • 作者: Dennis Lee
  • 机构: University of Waterloo, University of Pennsylvania
  • 链接: arXiv | PDF

  • 来源: 📄 arXiv 关键词检索

  • 核心方法: TimeSeek 评估 10 个前沿 LLM（含 GPT-5, DeepSeek）在 150 个 CFTC 监管的 Kalshi 二元市场上的预测可靠性。关键创新是时间维度——在预测市场生命周期的 5 个 checkpoint 分别测试，共 15000 个预测。区分有无 web search 的效果。

  • 锐评: Benchmark 论文，方向有意思但创新不大。用预测市场衡量 LLM 的"真实世界"推理能力是个好角度，但 150 个市场的样本量偏小。没有 BSS (Brier Skill Score) 的改进幅度说明，很难判断实际价值。 💤

5. Beyond Fluency: Toward Reliable Trajectories in Agentic IR

  • 作者: Anushree Sinha, Srivaths Ranganathan, Debanshu Das, Abhishek Dharmaratnakar
  • 机构: Google
  • 链接: arXiv | PDF

  • 来源: 📄 arXiv 关键词检索

  • 核心方法: 分析 Agentic IR（基于 agent 的信息检索）中的 trajectory 可靠性问题——agent 在多步 Reason-Act-Observe 循环中，早期小错误会级联放大，即使语言输出仍然流畅。提出 functional misalignment 的概念。

  • 锐评: Google 出品，问题定义不错（fluency ≠ reliability），但更像是 position paper / analysis 而非方法论文。对做 agent 系统的人有参考价值，但没有具体的解决方案。 💤
