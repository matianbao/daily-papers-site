Agent 框架与方法

1. GEMS: Agent-Native Multimodal Generation with Memory and Skills

  • 作者: Zefeng He, Siyuan Huang, Xiaoye Qu, Yafu Li, Tong Zhu, Yu Cheng, Yang Yang
  • 机构: Shanghai AI Laboratory, Nanjing University, SJTU, CUHK
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 66
  • 核心方法: 提出 agent-native 的多模态生成框架，三大组件：Agent Loop（多 agent 闭环迭代优化生成质量）、Agent Memory（层级化轨迹记忆，存储事实状态和压缩经验摘要）、Agent Skill（可扩展的领域技能库，按需加载）。让 6B 小模型 Z-Image-Turbo 超越 SOTA Nano Banana 2。
  • 对比方法/Baselines: 在 GenEval2、SpatialGenEval 等 5 个主流 + 4 个下游任务上评测，对比多种生成后端。
  • 借鉴意义: Agent Memory 的层级设计（事实状态 + 压缩经验）对做推荐系统的用户记忆建模有启发。Agent Skill 的按需加载思路可以借鉴到推荐系统的多场景策略调度。
  • 锐评: 这篇是 agent 在生成任务上的正确打开方式——不是简单套壳，而是把 memory 和 skill 设计得有层次。6B 模型超 SOTA 的结果说明 agent 框架确实能放大小模型能力。66 个 upvote 说明社区认可。质疑点：迭代优化的额外推理成本有多大？论文没给 latency 数据。 🔥

    • 2. OrgAgent: Organize Your Multi-Agent System like a Company

  • 作者: Yiru Wang, Xinyue Shen, Yaohui Han, Michael Backes, Pin-Yu Chen, Tsung-Yi Ho
  • 机构: CUHK, IBM Research, CISPA
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: 公司式层级 multi-agent 框架，分三层：治理层（规划+资源分配）、执行层（任务求解+审核）、合规层（最终答案控制）。将 multi-agent 推理分解为类企业组织结构。
  • 对比方法/Baselines: 对比了 flat multi-agent、single-agent baseline。在 SQuAD 2.0、MuSiQue、MuSR 上评测，使用 GPT-5mini / GPT-OSS-120B / LLaMA-3.1-8B。
  • 借鉴意义: 层级组织 + skill-based worker pool 的设计对构建复杂推荐/广告系统中的 agent 编排有直接参考价值。治理-执行-合规的三层设计可以映射到广告系统的预算-投放-风控架构。
  • 锐评: 数据说话：GPT-OSS-120B 上层级设置比 flat 提升 102.73%，token 消耗降低 74.52%——这不是 incremental improvement。组织结构确实对 multi-agent 系统的效率和效果都有显著影响。但实验只在 QA 类 benchmark 上做，是否能推广到更复杂的 open-ended 任务还需验证。 🔥

    • 3. Gen-Searcher: Reinforcing Agentic Search for Image Generation

  • 作者: Kaituo Feng, Manyuan Zhang, Shuang Chen 等
  • 机构: 未明确标注
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 51
  • 核心方法: 首个搜索增强图像生成 agent。通过多跳推理和搜索收集文本知识+参考图片，用于有知识基础的生成。构建了 Gen-Searcher-SFT-10k 和 Gen-Searcher-RL-6k 数据集，引入 KnowGen benchmark。用 SFT + 双奖励（文本+图像）的 GRPO 训练。
  • 对比方法/Baselines: 在 KnowGen 和 WISE 上评测，相比 Qwen-Image 提升约 16 和 15 分。
  • 借鉴意义: 搜索增强 + RL 训练 agent 的范式可以迁移到推荐场景：用搜索 agent 增强推荐理由生成、用 RL 优化搜索策略。
  • 锐评: "搜索增强生成"从 RAG 扩展到图像领域，方向有新意。双奖励的 GRPO 训练设计合理。51 个 upvote，社区反响不错。但全开源的承诺需要验证——数据集质量决定了这条路能走多远。 👀
  • 💡 想精读？ 运行：读一下 Gen-Searcher: Reinforcing Agentic Search for Image Generation

    4. Agent READMEs: An Empirical Study of Context Files for Agentic Coding

  • 作者: Worawalan Chatlatanagulchai 等
  • 机构: Nara Institute of Science and Technology, Queen's University, Kasetsart University
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 28
  • 核心方法: 对 1,925 个仓库中的 2,303 个 agent 上下文文件（CLAUDE.md、AGENTS.md 等）进行大规模实证分析。发现：开发者优先写功能性指令（构建命令 62.3%、实现细节 69.9%、架构 67.7%），但安全（14.5%）和性能（14.5%）等非功能需求严重缺失。
  • 借鉴意义: 如果你在用 Claude Code 的 CLAUDE.md，这篇告诉你大家都在写什么、哪些被忽略了。直接指导你优化自己的 agent 上下文文件。
  • 锐评: 实证研究做得扎实，2,303 个文件的规模够大。发现"安全和性能指令缺失"这个 gap 有实际意义——写 CLAUDE.md 的时候谁想过加安全约束？但这篇更像 MSR/ICSE 的 empirical study，方法论创新不多。 👀
  • 💡 想精读？ 运行：读一下 Agent READMEs: An Empirical Study of Context Files for Agentic Coding

    • 5. The AI Scientist-v2: Workshop-Level Automated Scientific Discovery

  • 作者: Yutaro Yamada, Robert Tjarko Lange, Cong Lu 等
  • 机构: Sakana AI（推测）
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 19
  • 核心方法: AI Scientist 的 v2 版本。去掉了对人类代码模板的依赖，使用 progressive agentic tree search + experiment manager agent。新增 VLM 反馈环优化图表质量。一篇全自动论文通过 ICLR workshop 同行评审。
  • 借鉴意义: Agentic tree search 的方法论可以借鉴到推荐系统的自动化实验调参。
  • 锐评: "第一篇 AI 全自动通过同行评审"是个里程碑式的标题，但仔细看是 workshop paper 不是主会。不过方向是对的——从 v1 的模板依赖到 v2 的自主搜索，进步明显。代码开源加分。 👀
