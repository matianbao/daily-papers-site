1. UniMixer: A Unified Architecture for Scaling Laws in Recommendation Systems

  • 作者: Mingming Ha, Guanchen Wang, Linxun Chen, Xuan Rao, Yuexin Shi, Tianbao Ma, Zhaojie Liu, Yunqian Fan, Zilong Lu, Yanan Niu, Han Li, Kun Gai
  • 机构: 未明确标注（从作者阵容和风格看大概率是快手/美团级别的工业界团队）
  • 链接: arXiv | PDF
  • 来源: 📰 HF Daily，⬆️ 1
  • 核心方法: 输入是推荐系统的特征 token 序列，输出是 CTR/UAUC 预测。关键创新是把 attention-based、TokenMixer-based 和 FM-based 三种主流 scaling 架构统一到一个 parameterized feature mixing 框架下。通过将 rule-based TokenMixer 等价变换为可学习的参数化结构，去掉了 heads 数量必须等于 token 数量的约束。还设计了轻量版 UniMixing-Lite 进一步压缩参数和 FLOPs。
  • 对比方法/Baselines: 对比了 RankMixer、HiFormer 等 SOTA scaling 模型。在 ~100M 参数量级做了全面对比，还有完整的 scaling curve 实验。
  • 借鉴意义: 直接相关。如果你在做推荐模型的 scaling，这篇统一框架值得细看。UniMixing-Lite 的压缩思路对在线 serving 有实际意义。有线上 A/B test 结果，不是纯学术灌水。
  • 锐评: 这篇有真东西。统一三种 scaling 范式的理论框架是有价值的，不是简单拼凑。线上 A/B test 是硬通货。但 HF upvotes 只有 1，说明社区关注度不高——推荐系统的论文在 HF 上天然冷门。质疑点：unified framework 的实际收益是否真的来自统一，还是单纯因为多了可学习参数？ablation 看起来做了但需要仔细验证。 🔥

    2. To Memorize or to Retrieve: Scaling Laws for RAG-Considerate Pretraining

  • 作者: Karan Singh 等
  • 机构: Stanford University, Carnegie Mellon University, Patronus AI, The Ohio State University
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: 输入是不同规模的预训练数据 + 检索语料，输出是模型在多个 benchmark 上的性能。提出三维 scaling framework，将性能建模为模型大小、预训练 token 数和检索语料大小的函数。训练 30M-3B 参数的 OLMo-2 模型，在 DCLM 数据上变化预训练数据量（1-150x）和检索库大小（1-20x）。核心发现：检索的边际效用强烈依赖模型规模、任务类型和预训练饱和程度。
  • 对比方法/Baselines: 对比了纯参数化 baseline（无检索）、不同检索库大小的 RAG 配置。使用 FAISS 索引，在 ARC、HellaSwag、SciQ 等 benchmark 上评测。
  • 借鉴意义: 对做大规模推荐系统有启发——推荐系统同样面临"记忆 vs 检索"的 tradeoff。这篇给出的定量框架可以类比到推荐场景中的 embedding table size vs 实时检索的资源分配问题。
  • 锐评: 实验规模扎实（30M 到 3B，100B token），三维 scaling manifold 是个有用的分析工具。但结论其实有点 obvious：大模型不那么需要检索，小模型更需要。真正有价值的是定量的 tradeoff curve。论文标题起得好，"To Memorize or to Retrieve" 抓住了核心问题。 👀

    • 3. MSA: Memory Sparse Attention for Efficient End-to-End Memory Model Scaling to 100M Tokens

  • 作者: Yu Chen 等
  • 机构: Peking University
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 44
  • 核心方法: 提出 Memory Sparse Attention，通过 scalable sparse attention + document-wise RoPE 实现训练和推理的线性复杂度。核心是将 memory 和 reasoning 解耦，配合 KV cache 压缩和 Memory Parallel 技术，在 2xA800 上实现 100M token 推理。还提出 Memory Interleaving 支持跨 memory 段的多跳推理。
  • 对比方法/Baselines: 对比了 RAG 系统、memory agent（如 MemAgent）、混合线性注意力方法、RWKV 等。在 NIAH、HotpotQA、NarrativeQA 等长上下文 benchmark 上评测。
  • 借鉴意义: 如果你在做长序列用户行为建模（比如用户历史行为 100M+ token 级别），MSA 的 sparse attention + memory 解耦思路可以参考。
  • 锐评: 100M token 的 scaling 确实 impressive，从 16K 到 100M 不到 9% 的性能下降是硬数据。但 44 个 upvote 说明社区买账。质疑点在于：document-wise RoPE 在非文档结构的序列上是否还能 work？实际部署时 2xA800 的门槛不低。 👀

    • 4. Universal YOCO for Efficient Depth Scaling

  • 作者: Yutao Sun 等
  • 机构: Microsoft Research, Tsinghua University
  • 链接: arXiv | PDF
  • 来源: 📰 HF Daily，⬆️ 9
  • 核心方法: 在 YOCO decoder-decoder 架构基础上引入 Universal Self-Decoder（参数共享的递归计算），将迭代过程限制在浅层 efficient-attention 层。实现常量 KV cache + 线性 prefilling + 深度可 scaling 的组合。输入/输出是标准语言模型任务。
  • 对比方法/Baselines: 对比了标准 YOCO、ParScale 等 depth scaling 方法。在数学 benchmark 和 NIAH 上评测。
  • 借鉴意义: KV cache 不随深度增长是个有实际意义的性质。对在线 serving 场景的推理成本控制有参考价值。
  • 锐评: 微软 + 清华的组合，YOCO 系列的延续。思路清晰：递归 + efficient attention 的组合确实 1+1>2。但 loop scaling 的 diminishing return 明显，从 loop 3 到 5 的提升很小。实际部署中 recursive computation 的延迟影响需要更多分析。 👀

    • 5. Spectral Compact Training: Pre-Training LLMs via Permanent Truncated SVD

  • 作者: Björn Roman Kohlberger
  • 机构: EctoSpace, Dublin
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: 用永久截断 SVD 分解 W = U diag(s) V^T 替代 dense 权重矩阵，全程不 materialize 完整矩阵。梯度直接在 compact spectral factors 上流动，每步用 QR 分解将 U/V 投回 Stiefel 流形。Rank 32 时每层 MLP 内存降低 199x。
  • 对比方法/Baselines: 对比了 LoRA、GaLore 等低秩方法。在 SmolLM2-1.7B 上做了 rank sweep 实验。
  • 借鉴意义: 如果你在资源受限环境下训练大模型，这个思路很极端但有启发。70B 模型在 Steam Deck 上跑完一个 training step 是个很好的 demo。
  • 锐评: 想法足够大胆——永远不 materialize dense matrix。但 loss floor 4.2-4.5 vs dense 的 1.29，差距巨大。作者说"所有 rank 收敛到同一 loss floor"，这其实是坏消息：说明瓶颈不在 rank 而在这种分解本身的表达力。目前只是 proof of concept，离实用还远。 ⚠️

    6. A Survey of On-Policy Distillation for Large Language Models

  • 作者: Mingyang Song, Mao Zheng
  • 机构: Tencent, China
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: 首篇全面综述 LLM 的 On-Policy Distillation（OPD）。提出统一 f-divergence 框架，沿三个维度分类：feedback signal（logit/outcome/self-play）、teacher access（white-box/black-box/teacher-free）、loss granularity（token/sequence/hybrid）。覆盖 MiniLLM、GKD、DistiLLM、DeepSeek-R1 蒸馏等方法。
  • 对比方法/Baselines: 综述性质，系统比较了 DPO、GRPO、REINFORCE 等方法在蒸馏框架下的定位。
  • 借鉴意义: 如果你在用大模型蒸馏小模型做推荐/广告的 serving，on-policy vs off-policy 的 tradeoff 直接相关。腾讯出品，工业视角应该有参考价值。
  • 锐评: 来自腾讯，对 OPD 的系统化做得不错。f-divergence 统一框架理论上 clean。但综述嘛，关键看你需不需要这个系统视角。如果你已经在用 DeepSeek-R1 的蒸馏方案，这篇帮你理解它在整个 landscape 中的位置。 👀

    7. Efficient Memory Management for LLM Serving with PagedAttention

  • 作者: Woosuk Kwon, Zhuohan Li 等
  • 机构: UC Berkeley（推测）
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 49
  • 核心方法: 经典 vLLM 论文。借鉴 OS 虚拟内存的 paging 思想管理 KV cache，实现近零内存浪费和灵活的跨请求 KV cache 共享。
  • 借鉴意义: 如果你还没读过 vLLM 的论文，值得补课。对 LLM serving 的基础设施设计有参考。
  • 锐评: 2023 年的经典论文又上 trending，大概是有新用户在补课。论文本身是 LLM serving 的基石之作，没什么好喷的。但放在"今日推荐"里有点奇怪——这是历史文献不是新论文。 💤

    • 8. Adaptive Parallel MCTS for Efficient Test-time Compute Scaling

  • 作者: Hongbeen Kim, Juhyun Lee 等
  • 机构: 未明确标注
  • 链接: arXiv | PDF
  • 来源: 📄 arXiv 关键词检索
  • 核心方法: 提出 negative early exit 机制剪枝 MCTS 中无进展的搜索轨迹，配合 adaptive boosting 将回收的计算资源重新分配。集成到 vLLM 中，降低 p99 延迟同时提升吞吐量。
  • 对比方法/Baselines: 对比了标准 MCTS、带 positive early exit 的 MCTS、beam search。使用 EurusPRM 作为 reward model。
  • 借鉴意义: test-time compute scaling 是当前 LLM 推理的热点。如果你在部署需要 reasoning 的 LLM 服务，这篇的延迟优化思路有参考价值。
  • 锐评: 工程导向的论文，negative early exit 的思路简单有效。p99 延迟的优化对生产环境确实重要。但创新性有限——early stopping + 资源重分配在其他领域早就有了。 🫠

    • 9. Bitnet.cpp: Efficient Edge Inference for Ternary LLMs

  • 作者: Jinheng Wang 等
  • 机构: Peking University, Microsoft Research
  • 链接: arXiv | PDF
  • 来源: 🔥 HF Trending，⬆️ 17
  • 核心方法: 针对 BitNet b1.58 优化的推理系统，核心是 Ternary Lookup Table (TL) 和 Int2 with Scale (I2_S) 两种 mpGEMM 方案，实现 sub-2-bit 无损推理。

  • 锐评: 6.25x 加速的数字好看，但 ternary LLM 本身的精度还没有被广泛验证。方向有意思但赛道太窄。 💤
